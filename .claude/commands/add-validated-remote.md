description: Создать RemoteEvent строго с серверной middleware-валидацией
argument-hint: <RemoteName> <описание payload и допустимых значений>
---

Создай новый RemoteEvent `$ARGUMENTS` строго по правилам `docs/02`:

1. Объяви remote в `src/shared/Net/RemoteSchema` (единый реестр).
2. Серверный обработчик ОБЯЗАН пройти middleware ДО любой логики:
   - type-assert каждого аргумента;
   - проверка по серверному whitelist допустимых значений;
   - bounds для числовых входов;
   - rate-limit на игрока.
3. Вся мутация состояния — только после успешной валидации. При провале — лог + тихий игнор (не кик).
4. Клиент использует remote только для намерения/ввода; UI обновляется оптимистично, истина — от сервера.
5. Напиши TestEZ-спеку: отправь невалидные/мусорные значения и проверь, что сервер их отвергает.

Не используй RemoteFunction:InvokeClient(). Для server->client — RemoteEvent.
