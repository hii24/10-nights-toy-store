# Шаблон промпта рабочей сессии (каждая последующая сессия)

Начинай каждую новую сессию с команды `/session-start` (она выполнит ритуал §4.2). Если команд нет под рукой — используй промпт ниже.

---

Начни рабочую сессию по методологии (`CLAUDE_CODE_METHODOLOGY.md` §4):

1. `pwd`; прочитай `harness/claude-progress.md` и `git log --oneline -20`.
2. Прочитай `harness/feature_list.json`; выбери ОДНУ фичу `passes:false` высшего приоритета для текущей фазы.
3. Запусти `harness/init.sh` и сделай смоук-тест. Если сломано — сначала почини.
4. В plan mode (Opus) предложи план реализации выбранной фичи. Жди моего «ок».
5. Реализуй (Sonnet) по правилам `docs/`. RemoteEvent — только через `/add-validated-remote`. Вся логика геймплея — серверная.
6. TestEZ + end-to-end плейтест в Studio. Пометь `passes:true` ТОЛЬКО после end-to-end.
7. Прогони `/review` (минимум security-reviewer). Заверши через `/ship` (lint, тесты, коммит, запись прогресса).

Одна фича за сессию. Только текущая фаза. Оставь код в состоянии «годно к мёрджу».
