# 07 — Стандарты кода

## Стиль
- Официальный **Roblox Lua Style Guide** (спроектирован по образцу Google C++ Style Guide).
- **Selene** (линтер) + **StyLua** (форматтер) — обязательны, висят на hooks (авто после правок) и в CI.
- Strict Luau typing где возможно; типы — в `src/shared/Types`.

## Конвенции
- Сервисы Knit: один файл = один сервис, имя `XxxService`. Контроллеры клиента — `XxxController`.
- Нейминг: PascalCase для модулей/сервисов/классов; camelCase для локальных; SCREAMING_SNAKE для констант.
- Guard clauses вместо вложенных if. Ранний возврат.
- Никаких «магических чисел» в логике — выносить в `src/shared/Config`.
- Очистка ресурсов: Janitor/Trove для соединений/инстансов (без утечек).
- RemoteEvent создаётся ТОЛЬКО через хелпер с middleware-валидацией (см. `docs/02` и команду `/add-validated-remote`).

## Комментарии и доки
- Комментировать «почему», а не «что». Публичные функции модулей — с краткой докстрокой (стиль Moonwave).
- Каждый сервис — с шапкой: зона ответственности, что владеет, какие remote'ы экспонирует.

## Тесты
- Каждый серверный сервис с нетривиальной логикой -> TestEZ-спека в `tests/`.
- Тесты не редактируются ради «прохождения» (см. методологию §4).
